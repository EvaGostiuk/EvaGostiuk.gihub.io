Log # 04

While exploring ChatGPT, I encountered description of interesting technique. Here is accurate description (generated by ChatGPT) for steps of the technique:

Implementing Latent Semantic Analysis (LSA) typically involves the following steps:

  1. Preprocessing the data: This includes cleaning and preprocessing the text data, such as removing stop words, stemming or lemmatizing, and removing any other irrelevant words or symbols.
  2. Creating a term-document matrix: This matrix represents the co-occurrence of words in a corpus of documents, with rows representing words and columns representing documents. The entries in the matrix are typically the term frequency (TF) or term frequency-inverse document frequency (TF-IDF) values of the words in the documents.
  3. Applying singular value decomposition (SVD) to the term-document matrix: SVD is used to reduce the dimensionality of the matrix by finding a low-rank approximation of it. The number of latent concepts to retain is determined by the number of non-zero singular values.
  4. Extracting the latent semantic structure: The columns of the matrix obtained from SVD are the latent semantic structure, which can be used to find similarities between documents based on the underlying topics they cover.
  5. Document comparison: The resulting matrix can be used to compare documents by calculating the cosine similarity between the columns of the matrix, which correspond to the documents.

Log # 09

Ok. I revisited Nielsen’s Heristics and made actionable checklist for each heristic. (They are in collapsed state, so click on each labeled area to expand corresponding list)

Visibility of System Status (what’s happening now?)
  • Provide feedback for every action performed which indicates what action was taken and what the system is currently doing.
  • Show loading indicators which should give a sense of how long a process will take and provide progress updates.
  • If the system is in a particular state, clearly label that state so the user can easily understand what is happening.
  • If an error occurs, clearly communicate the problem and provide guidance on how to fix it.
  • Use colors and icons to provide visual cues to make it easy to understand the status of the system at a glance.
  • Use animations to show changes in system status.
  • Make sure the status information is always visible.
  • Test the system with real users to ensure the visibility of system status is effective.
Match between system and the real world
  • Use metaphor/analogy that helps to understand system in a real-world context.
  • Follow real-world conventions and make information appear in a natural and logical order.
  • Provide visual aids, such as icons, that help users recognize and understand the functionality of the system.
User Control and Freedom (“Undo”; What’s next?)
  • Provide clear and visible navigation and interaction options
  • Provide clearly labeled undo and redo options.
  • Include option that allow users to safely exit the process.
  • Avoid forcing users to perform actions in a very specific order. Instead:
    • offer multiple paths to complete same task,
    • allow users to save and resume progress at any point,
    • allow users to temporarily skip some steps and return to them later
  • Provide keyboard shortcuts and other alternatives to mouse-based interactions.
  • Provide a clear and easy-to-use interface which allows users to customize and personalize their experience, such as
    • choosing their preferred language,
    • choosing their preferred font size/style,
    • choosing their preferred colour scheme,
    • disable auto-play,
    • customize auto-save options,
    • allow to customize keyboard shortcuts for all actions

Error Prevention (confirmation options)
  • Clearly indicate which actions are reversible and which are irreversible.
  • Prompt users for confirmation before carrying out irreversible actions.
  • Use input validation to prevent users from entering incorrect or invalid data.
  • Provide tips/examples to prevent common errors and misunderstandings.
  • Use default values and auto-complete fields.
Error handling: recognition, diagnosis and recovery
  • Provide clear, concise, and non-technical(simple) error message that explains the problem.
  • Indicate the severity of the error.
  • Provide information about how error will affect the user's work.
  • Provide guidance on how to prevent the error from occurring again in the future.
  • Provide alternative paths or solution that would help to recover from the error.
  • Provide a way for users to report errors.
  • Save and analyze error logs to identify and fix any recurring issues or trends in the system.
  • Create error handling system that catches all possible errors and provides appropriate feedback to the user.
Consistency and standards (of words and images)
  • Use consistent terminology and labels throughout the interface.
  • Use consistent placement of interface elements such as buttons, menus, and icons.
  • Ensure that visual elements (such as colors, fonts,icons, and other) are consistent throughout the experience
  • Ensure that error messages and warning messages are consistent in wording, style, tone, location.
  • Follow established industry/platform design and usability guidelines.
  • Follow established accessibility guidelines
Recognition vs Recall (users not forced to remember much/anything)
  • Provide relevant and helpful hints or instructions at the appropriate places.
  • Provide familiar and consistent patterns.
  • Utilize common and widely accepted conventions.
  • Make it easy for users to spot important information and actions quickly.
  • Make next related actions visible and easily accessible.
  • Allow user to see/review list of previous actions.
  • Implement Consistency and standards.
Flexibility and efficiency (tailoring, response time)
  • Provide keyboard shortcuts and other accelerators (such as auto-fill)
  • Allow users to work with multiple items at once.
  • Use automation and smart defaults to reduce repetitive or tedious tasks.
  • Make frequent actions more visible/accesible.
  • Provide options for customization and personalization, so users can adjust the interface to their preferences and needs.
  • Allow users to save and resume tasks even if they need to switch devices.
Aesthetic and minimalist design
  • Minimize the use of images and other non-essential elements.
  • Use icons and other visual elements only when they add value and aid in recognition.
  • Use a consistent visual hierarchy to make important information stand out.
  • Use whitespace to guide the user's attention.
  • Choose a simple/small color palette.
  • Use animation sparingly and only when it serves a specific purpose.
  • Use contrast to make important elements stand out.
Help and documentation (user-appropriate)
  • Provide clear, concise, and simple instructions on how to use the system.
  • Provide contextual help that is easily accessible.
  • Provide relevant and up-to-date documentation.
  • Use visual aids such as screenshots and videos.
  • Provide contact information for support in case users need additional help.
Log # 10

Way of passing information between child and parent components:

React_Context

In parent:

import {useState, createContext} from "react";
import Child from "./Child"

export const reactContext = createContext();

const Parent = () => {
	const [val, setVal] = useState('default');
	
	return (
		<reactContext.Provider value={{ val, setVal }}>
		    <Child />
		</reactContext.Provider>
	);
}

In child:

import {useContext} from "react";

const Child = () => {
	const [val, setVal] = useContext(reactContext);
	
	return (
		<div> 
			<p> {val} </p>
		</div>
	);
}

export default Child;
